## 一、通讯基础参数（与M109E完全一致）

| 参数 | 值 |
|------|-----|
| 接口 | RS-485 / TTL |
| 波特率 | 9600bps |
| 数据位 | 8位 |
| 停止位 | 1位 |
| 校验 | 无 |
| 流控 | 无 |
| 模式 | 主从（主机=上位机，从机=货道板） |

## 二、帧格式（20字节定长，与M109E完全一致）

### 2.1 指令帧（主机→从机）

```
字节:  [1字节]    [1字节]    [16字节]         [2字节]
       从机地址    指令       Y1-Y16 数据       CRC16
       (1-8)      (见指令表)  (不足补0)         (低字节在前)
```

### 2.2 响应帧（从机→主机）

```
字节:  [1字节]    [1字节]    [16字节]         [2字节]
       主机地址    指令       Z1-Z16 数据       CRC16
       (0x00)     (同指令)   (不足补0)         (低字节在前)
```

### 2.3 校验方式

```
CRC16-MODBUS
Poly:  0x8005
Init:  0xFFFF
校验代码低字节在前、高字节在后
```

### 2.4 通讯流程

```
i.   主机下发指令
ii.  等待 50ms
iii. 接收从机响应（超时 1 秒）

规则：
- 每次通讯由主机发起请求，从机应答
- 主机收到上次请求的应答或者超时后才可以发起新的请求
- 从机地址 1~8，主机地址为 0
- 地址 255 为广播地址（仅用于修改从机地址）
- 指示灯连续闪烁次数 = 通讯地址
```

## 三、指令表（与M109E完全一致）

### 3.1 获取序列号（ID）— 指令 0x01

**请求：**
```
[从机地址] [01] [00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00] [CRC]
```

**响应：**
```
[00] [01] [Z1-Z12=控制卡ID号] [00 00 00 00] [CRC]
```

**示例：**
```
发→ 01 01 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 71 88
收← 00 01 00 64 00 3B 04 47 36 32 33 38 36 39 00 00 00 00 6F 8E
```

---

### 3.2 查询电机运行状态（Motor Poll）— 指令 0x03

**请求：**
```
[从机地址] [03] [00×16] [CRC]
```

**响应：**
```
[00] [03] [Z1] [Z2] [Z3] [Z4-Z5] [Z6-Z7] [Z8-Z9] [Z10] [00×6] [CRC]
```

| 字节 | 含义 | 值域 |
|------|------|------|
| Z1 | Run指令执行状态 | 0=空闲, 1=执行中, 2=执行完毕 |
| Z2 | 正在运转的电机编号 | 00~99 (0x00~0x63)，共100个 |
| Z3 | 执行结果 | 见错误码表 |
| Z4-Z5 | 峰值电流 | 单位mA，高字节在前 |
| Z6-Z7 | 平均电流 | 单位mA，高字节在前 |
| Z8-Z9 | 运行时间 | 单位ms，高字节在前 |
| Z10 | 光幕状态 | 0=无掉货, 1~200=货物经过光栅的时间(ms) |

**错误码表（Z3）：**
| 代码 | 含义 |
|------|------|
| 0x00 | 成功 |
| 0x01 | 过流、负载过重、或者卡货 |
| 0x02 | 欠流。一般是电线断开、或者负载未安装 |
| 0x03 | 超时，超过7秒仍未检测到电机到位信号 |
| 0x04 | 光幕自检失败，未启动电机 |
| 0x05 | 有反馈电磁铁门未开 |
| 0x0A | 三线制电机通电1.5秒后，微动开关仍未被按下 |

**示例：**
```
发→ 01 03 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 D0 E8
收← 00 03 02 00 00 05 BA 00 77 0A 3C 00 00 00 00 00 00 00 98 46

解析：
  Z1   = 0x02   → 运转完毕
  Z2   = 0x00   → 电机编号 0
  Z3   = 0x00   → 无故障（成功）
  Z4-Z5= 0x05BA → 峰值电流 1466mA
  Z6-Z7= 0x0077 → 平均电流 119mA
  Z8-Z9= 0x0A3C → 运转耗时 2620ms
```

### 3.3 启动电机（Motor Run）— 指令 0x05

**请求：**
```
[从机地址] [05] [Y1] [Y2] [Y3] [Y4] [Y5] [Y6] [Y7] [00×9] [CRC]
```

| 字节 | 含义 | 值域 |
|------|------|------|
| Y1 | 电机索引号 | 00~59 |
| Y2 | 电机类型 | 见电机类型表 |
| Y3 | 光幕工作模式 | 0/1/2（见下方说明） |
| Y4 | 检测微动延时 | 2-50，对应0.2秒~5秒，默认15(1.5秒) |
| Y5 | 保留 | 0x00 |
| Y6 | 电机到位超时时间 | 1-250，代表0.1S~25.0S，0=默认70(7秒) |
| Y7 | 时间锁动作时间 | 1-100，代表0.1S~10.0S，其他=默认2(0.2S) |

**电机类型表（Y2）：**
| 值 | 类型 |
|----|------|
| 0x00 | 无反馈电磁铁 |
| 0x01 | 两线有反馈电磁铁（不建议使用） |
| 0x02 | 两线制电机 |
| 0x03 | 三线制电机 |
| 0x04 | 12秒电插锁 |
| 0x05 | 共正型两线电机 |
| 0x06 | 皮带货道；信号 H→L→H |
| 0x07 | 皮带货道；信号 L→H→L |
| 0x08 | 时间控制型电磁锁 |
| 0x09 | 三线有反馈电磁锁 |
| 0x0A | L10~L1, C10~C1, H8~H1 2WIRE |
| 0x0B | L10~L1, C10~C1, H8~H1 3WIRE |
| 0x0C | L10~L1 无反馈电磁铁 |
| 0x0D | 共正型无反馈电磁锁 |
| 0x0E | 皮带货道；信号 H→L→H |

**光幕工作模式（Y3）：**
| 值 | 模式 |
|----|------|
| 0 | 电机运转不参考光幕 |
| 1 | 启动电机前，需光幕自检成功；电机到位后停止 |
| 2 | 启动电机前，需光幕自检成功；光幕检测到掉货后电机立即停止，未检测到掉货时电机一直运转直到超时（7s） |

**响应：**
```
[00] [05] [Z1] [00×15] [CRC]
```

| Z1 | 含义 |
|----|------|
| 0x00 | 已启动 |
| 0x01 | 无效的电机索引号 |
| 0x02 | 另一台电机在运行 |
| 0x03 | 上次运转的结果未清除 |

**示例（启动#0号马达，三线制，无光幕）：**
```
发→ 01 05 00 03 00 00 00 00 00 00 00 00 00 00 00 00 00 00 70 48
收← 00 05 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 62 B5
```

**示例（启动#0号马达，三线制，光幕优先）：**
```
发→ 01 05 00 03 02 00 00 00 00 00 00 00 00 00 00 00 00 00 73 8A
收← 00 05 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 62 B5
```

**示例（时间控制格子锁带光幕）：**
```
发→ 01 05 00 08 01 00 00 01 00 00 00 00 00 00 00 00 00 00 3F 72
    08=特殊格子锁模式  01=有光幕  01=100ms(范围01d~250d)
```

## 四、30×30板与M109E的通道映射（平替方案）

### 4.1 核心原则

**协议层100%兼容M109E**，不修改任何帧格式、指令码、数据字段含义。

### 4.2 通道映射方案

M109E的电机索引号为 **00~99**（单字节），对应10×10矩阵的100个通道。

30×30板有900个通道，映射方式：

```
M109E通道号 (Y1/Z2)  →  30×30矩阵位置

通道 0  = Row 0, Col 0    (PM_OUT1, NM_OUT1)
通道 1  = Row 0, Col 1    (PM_OUT1, NM_OUT2)
...
通道 29 = Row 0, Col 29   (PM_OUT1, NM_OUT30)
通道 30 = Row 1, Col 0    (PM_OUT2, NM_OUT1)
...
通道 59 = Row 1, Col 29   (PM_OUT2, NM_OUT30)
通道 60 = Row 2, Col 0    (PM_OUT3, NM_OUT1)
...
通道 99 = Row 3, Col 9    (PM_OUT4, NM_OUT10)
```

**前100个通道（0-99）= 前4行的前10列**，与M109E完全兼容。

### 4.3 扩展通道（100-899）的访问方式

对于超出M109E协议范围的通道（100-899），有两种扩展方案（待与上位机团队协商）：

#### 方案A：利用Y2字段（推荐，不影响M109E兼容性）

M109E的0x05指令中Y2是"电机类型"，但实际上电机类型由上位机配置。
可以在不破坏兼容性的前提下，用Y2的高位来扩展通道号：

```
实际通道号 = Y1 + (Y2高4位) × 100

其中 Y2 低4位 = 电机类型（0x00-0x0E，只需4bit）
     Y2 高4位 = 通道号扩展（0-8）

示例：
  Y1=50, Y2=0x03 → 通道50, 类型=三线制电机（兼容M109E）
  Y1=50, Y2=0x13 → 通道150, 类型=三线制电机（扩展）
  Y1=50, Y2=0x23 → 通道250, 类型=三线制电机（扩展）
  ...
  Y1=50, Y2=0x83 → 通道850, 类型=三线制电机（扩展）
```

#### 方案B：每板100通道，9块板级联

```
板卡地址1: 通道 0-99
板卡地址2: 通道 100-199
板卡地址3: 通道 200-299
...
板卡地址9: 通道 800-899

每块板使用独立地址，上位机按地址分发指令。
```

> **建议**：先用方案B（最简单，零协议改动），后续如需单板访问全部900通道再考虑方案A。

## 五、本板与M109E的硬件差异（协议层需适配）

| 项目 | M109E | 本板 | 协议层处理 |
|------|-------|------|-----------|
| 通道数 | 100 (10×10) | 900 (30×30) | 前100通道兼容，扩展方案见第四章 |
| 电机索引 | Y1=00~59 | Y1=00~59 | **保持不变** |
| 开关量输出 | 5路 (DO0-DO4) | 4路 (DO0-DO3) | DO4-DO6指令返回0xF0（无效） |
| 开关量输入 | 4路 (DI0-DI3) | 4路 (DI0-DI3) | **保持不变** |
| 温度 | NTC 1路 | NTC 1路 | **保持不变** |
| 温湿度 | DHT11 | 无DHT11 | Z1=0, Z2=温度, Z3=1（非最新） |
| 光幕 | 1路 | 1路（IN11） | **保持不变** |
| 到位检测 | H1-H10 (10路) | IN1-IN10 (10路) | **保持不变** |

## 六、CRC16-MODBUS 参考实现

```c
uint16_t crc16_modbus(uint8_t *data, uint16_t length)
{
    uint16_t crc = 0xFFFF;
    for (uint16_t i = 0; i < length; i++) {
        crc ^= data[i];
        for (uint8_t j = 0; j < 8; j++) {
            if (crc & 0x0001) {
                crc = (crc >> 1) ^ 0xA001;  // 0xA001是0x8005的反转
            } else {
                crc >>= 1;
            }
        }
    }
    return crc;  // 低字节在前，高字节在后
}
```
