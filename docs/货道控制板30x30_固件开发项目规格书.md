## 一、项目背景

### 1.1 目标
在已有30×30货道控制板硬件基础上，开发配套固件程序，实现900通道出货机构的驱动控制。

### 1.2 对标产品
- **参照产品**：M109E售货机控制卡（金华苏夏信息技术有限公司）
- **参照协议**：M109E通讯协议文档（已归档）

### 1.3 设计原则
- **协议向下兼容**：M109E上位机可直接控制本板（100通道子集）
- **裸机架构**：不使用RTOS，状态机驱动
- **资源精简**：适配STM32F103C8T6的64KB Flash / 20KB RAM

### 2.5 595级联顺序

```
P端（行/正极）级联链：
  MCU → U4(Q24-Q31) → U3(Q16-Q23) → U2(Q8-Q15) → U1(Q0-Q7)
  发送顺序：先发U4数据，最后发U1数据（MSB先发）

N端（列/负极）级联链：
  MCU → U27(Q24-Q31) → U26(Q16-Q23) → U??(Q8-Q15) → U5(Q0-Q7)
  发送顺序：先发U27数据，最后发U5数据

对称结构：P端4片=32位（用30位），N端4片=32位（用30位）
```

### 3.2 F02 — 74HC595矩阵驱动（详细）

#### 3.2.1 功能描述
通过7片74HC595级联，实现30行×30列矩阵的独立选通，控制900个通道的电机驱动。

#### 3.2.2 接口定义

```c
/**
 * @brief 初始化595控制引脚，全部输出低电平
 */
void HC595_Init(void);

/**
 * @brief 选通指定行（P端595）
 * @param row 行号 0-29，0xFF=全关
 * @note  同一时刻只能选通一行（或全关）
 *        选通=输出高电平，通过SS8050+4953驱动正极
 */
void HC595_SelectRow(uint8_t row);

/**
 * @brief 选通指定列（N端595）
 * @param col_mask 30位列的位掩码，bit0=列0...bit29=列29
 *                  1=选通（输出低，AO4812导通），0=断开
 * @note  可同时选通多列（一个电机或多电机同时驱动）
 */
void HC595_SetColumns(uint32_t col_mask);

/**
 * @brief 关闭全部输出（安全状态）
 */
void HC595_AllOff(void);
```

#### 3.2.3 时序要求
- SHCP（移位时钟）频率：1MHz ~ 4MHz（推荐2MHz）
- STCP（锁存脉冲）宽度：≥50ns
- MR（复位）脉冲宽度：≥100ns
- 移位完成后必须锁存（STCP上升沿）

#### 3.2.4 验收标准
- [ ] 能独立点亮30×30矩阵中任意一个交叉点的LED
- [ ] 能同时选通多列
- [ ] 全关操作后所有输出为低
- [ ] 移位无错位（连续切换不同行列，输出正确）

### 3.3 F03 — 电机控制（详细）

#### 3.3.1 电机类型定义（兼容M109E）

| 类型码 | 名称 | 驱动方式 | 到位检测 |
|--------|------|---------|---------|
| 0x00 | 无反馈电磁铁 | P高+N低，固定时间 | 无 |
| 0x01 | 两线有反馈电磁铁 | P高+N低 | H信号 |
| 0x02 | 两线制电机 | P高+N低 | H信号（到位停） |
| 0x03 | 三线制电机 | P高+N低 | H信号（到位+运行中） |
| 0x04 | 12秒电插锁 | P高+N低，12秒超时 | 无 |
| 0x05 | 共正型两线电机 | 同0x02 | H信号 |
| 0x06 | 皮带货道 H→L→H | P高+N低 | H信号变化 |
| 0x07 | 皮带货道 L→H→L | P高+N低 | H信号变化 |
| 0x08 | 时间控制型电磁锁 | P高+N低，可配时间 | 无 |
| 0x09 | 三线有反馈电磁锁 | P高+N低 | H信号 |
| 0x0A | 反向两线（L10→L1） | 同0x02，引脚反序 | H信号 |
| 0x0B | 反向三线（L10→L1） | 同0x03，引脚反序 | H信号 |
| 0x0C | 反向无反馈电磁铁 | 同0x00，引脚反序 | 无 |
| 0x0D | 共正型无反馈电磁锁 | 同0x00 | 无 |
| 0x0E | 皮带货道 V2 | P高+N低 | H信号变化 |

> 注：类型0x0A-0x0C的"反向"在本板中不适用（595驱动无方向概念），可统一为0x02/0x03/0x00。

#### 3.3.2 出货流程状态机

```
                    ┌─────────┐
                    │  IDLE   │ ← 上位机查询结果后回到IDLE
                    └────┬────┘
                         │ 收到启动指令(0x05)
                         ▼
                    ┌─────────┐
                    │ PRECHECK│ 光幕自检（如需要）
                    └────┬────┘
                         │ 自检通过
                         ▼
                    ┌─────────┐
              ┌────→│ RUNNING │ 选通行列，启动电机
              │     └────┬────┘
              │          │ 到位/超时/光幕触发
              │          ▼
              │     ┌─────────┐
              │     │STOPPING │ 关闭驱动，记录结果
              │     └────┬────┘
              │          │
              │          ▼
              │     ┌─────────┐
              └─────┤  DONE   │ 等待上位机读取
                    └─────────┘
```

#### 3.3.3 接口定义

```c
/**
 * @brief 电机控制模块初始化
 */
void Motor_Init(void);

/**
 * @brief 启动指定通道的电机
 * @param row       行号 0-29
 * @param col       列号 0-29
 * @param type      电机类型 0x00-0x0E
 * @param laser_mode 光幕模式 0/1/2
 * @param detect_delay 微动检测延时 2-50 (×0.1s)
 * @param timeout   到位超时 1-250 (×0.1s)，0=默认7s
 * @param lock_time 时间锁动作时间 1-100 (×0.1s)
 * @return 0=已启动, 1=无效通道, 2=电机运行中, 3=上次结果未清除
 */
uint8_t Motor_Start(uint8_t row, uint8_t col, uint8_t type,
                    uint8_t laser_mode, uint8_t detect_delay,
                    uint8_t timeout, uint8_t lock_time);

/**
 * @brief 查询当前电机运行状态
 * @param status    输出：0=空闲, 1=执行中, 2=执行完毕
 * @param channel   输出：当前运行通道号 (row*30+col)
 * @param result    输出：执行结果（错误码）
 * @param peak_cur  输出：峰值电流(mA)
 * @param avg_cur   输出：平均电流(mA)
 * @param run_time  输出：运行时间(ms)
 * @param laser_cnt 输出：光幕遮挡时间(ms)
 */
void Motor_GetStatus(uint8_t *status, uint16_t *channel,
                     uint8_t *result, uint16_t *peak_cur,
                     uint16_t *avg_cur, uint16_t *run_time,
                     uint8_t *laser_cnt);

/**
 * @brief 主循环调用，驱动电机状态机
 * @note  每1ms调用一次
 */
void Motor_Process(void);
```

#### 3.3.4 执行结果码（兼容M109E）

| 代码 | 含义 | 判定条件 |
|------|------|---------|
| 0x00 | 成功出货 | 到位信号有效 |
| 0x01 | 过流/负载过重/卡货 | 电流>阈值 |
| 0x02 | 欠流/断线/负载未接 | 电流<阈值 |
| 0x03 | 超时 | 超过timeout未到位 |
| 0x04 | 光幕自检失败 | 光幕模式1/2，自检时有遮挡 |
| 0x05 | 有反馈电磁铁门未开 | 类型0x01/0x09 |
| 0x0A | 微动开关未按下 | 三线制1.5s后H信号未变化 |

#### 3.3.5 验收标准
- [ ] 14种电机类型均能正确驱动
- [ ] 启动→到位→停止→返回结果 完整流程正确
- [ ] 过流/欠流/超时 故障码正确返回
- [ ] 光幕模式0/1/2 行为正确
- [ ] 连续出货100次无错位/误触发

### 3.4 F04 — RS-485通讯协议（详细）

#### 3.4.1 物理层
- 接口：USART1 → TP8485E-SR
- 波特率：9600bps
- 数据格式：8数据位，1停止位，无校验
- 总线模式：主从（主机=上位机，从机=货道板）
- 超时：主机等待响应超时1秒

#### 3.4.2 帧格式（兼容M109E）

```
指令帧（主机→从机），20字节：
┌──────┬──────┬──────────────────────┬──────────┐
│ 地址  │ 指令  │ Y1-Y16 数据（16字节）  │ CRC16    │
│ 1B   │ 1B   │ 16B                   │ 2B       │
└──────┴──────┴──────────────────────┴──────────┘

响应帧（从机→主机），20字节：
┌──────┬──────┬──────────────────────┬──────────┐
│ 地址  │ 指令  │ Z1-Z16 数据（16字节）  │ CRC16    │
│ 1B   │ 1B   │ 16B                   │ 2B       │
└──────┴──────┴──────────────────────┴──────────┘

CRC16-MODBUS: Poly=0x8005, Init=0xFFFF, 低字节在前
数据段不足16字节时补0
```

#### 3.4.3 通道编址方案

```
M109E兼容模式（Y1单字节）:
  Y1 = 通道号 (0-99)
  Y2 = 0

扩展模式（Y1+Y2双字节）:
  Y1 = 通道号低字节
  Y2 = 通道号高字节
  通道号 = Y1 + Y2×256 (范围 0-899)

  或者用行列方式（待定，建议与开发人员讨论选择）:
  Y1 = 行号 (0-29)
  Y2 = 列号 (0-29)
  通道号 = 行号 × 30 + 列号
```

> ⚠️ **待决策**：通道编址用"线性编号"还是"行列编号"？建议线性编号，与M109E更兼容。

#### 3.4.4 指令详细定义

##### 0x01 — 获取序列号

```
请求: [addr][01][00×16][CRC]
响应: [00][01][Z1-Z12=序列号][00×4][CRC]
```

##### 0x03 — 查询电机运行状态

```
请求: [addr][03][00×16][CRC]
响应: [00][03][Z1][Z2-Z3][Z4][Z5-Z6][Z7-Z8][Z9-Z10][Z11][00×5][CRC]

Z1:     运行状态 (0=空闲, 1=执行中, 2=完成)
Z2-Z3:  通道号 (16bit, 高字节在前)
Z4:     执行结果 (错误码)
Z5-Z6:  峰值电流 mA
Z7-Z8:  平均电流 mA
Z9-Z10: 运行时间 ms
Z11:    光幕遮挡时间 ms
```

##### 0x05 — 启动电机

```
请求: [addr][05][Y1][Y2][Y3][Y4][Y5][Y6][Y7][00×9][CRC]

Y1:     通道号低字节 (0-255)
Y2:     通道号高字节 (0=通道0-255, 1=256-511, 2=512-767, 3=768-899)
Y3:     电机类型 (0x00-0x0E)
Y4:     光幕模式 (0=不参考, 1=自检+到位停, 2=自检+掉货停)
Y5:     微动检测延时 2-50 (×0.1s)，默认15(1.5s)
Y6:     保留
Y7:     到位超时 1-250 (×0.1s)，0=默认70(7s)
Y8:     时间锁动作时间 1-100 (×0.1s)

响应: [00][05][Z1][00×15][CRC]
Z1: 启动结果 (0=已启动, 1=无效通道, 2=其他电机运行中, 3=上次结果未清除)
```

##### 0x07 — 读温度

```
请求: [addr][07][00×16][CRC]
响应: [00][07][Z1-Z2][00×14][CRC]

Z1-Z2: 温度值 (有符号16bit, ×0.1℃)
       例: 0x01F4 = 500 = 50.0℃
           0xFE0C = -500 = -50.0℃
       未接传感器返回 -50.0℃ (0xFE0C)
```

##### 0x08 — 开关量输出

```
请求: [addr][08][Y1][Y2][00×14][CRC]
Y1: DO索引 (0-3)
Y2: 操作 (1=ON, 0=OFF)

响应: [00][08][Z1][Z2][00×14][CRC]
Z1: DO索引
Z2: 操作结果 (Y2+0xF0)
```

##### 0x09 — 读开关量输入

```
请求: [addr][09][00×16][CRC]
响应: [00][09][Z1-Z4][00×12][CRC]

Z1-Z4: DI1-DI4状态 (1=外部连接, 0=断开)
```

##### 0x0B — 开停光幕

```
请求: [addr][0B][Y1][00×15][CRC]
Y1: 1=开, 0=关

响应: [00][0B][Z1][00×15][CRC]
Z1: 操作结果
```

##### 0x0C — 读光幕状态

```
请求: [addr][0C][00×16][CRC]
响应: [00][0C][Z1][Z2][00×14][CRC]

Z1: 光幕1状态 (1=有遮挡, 0=无遮挡) — INPUT_11
Z2: 光幕2状态 (1=有遮挡, 0=无遮挡) — INPUT_12
```

##### 0x0D — 读光幕遮挡计时

```
请求: [addr][0D][00×16][CRC]
响应: [00][0D][Z1][Z2][00×14][CRC]

Z1: 光幕1遮挡毫秒数
Z2: 光幕2遮挡毫秒数
注: 光幕开停时计数归零
```

##### 0x10 — 读温湿度

```
请求: [addr][10][00×16][CRC]
响应: [00][10][Z1][Z2][Z3][00×13][CRC]

Z1: 湿度 (本板无DHT11，固定返回0)
Z2: 温度 (同0x07指令)
Z3: 读取失败指示 (固定返回1，表示非最新采样)
```

##### 0x2A — 读单通道到位开关

```
请求: [addr][2A][Y1][Y2][00×14][CRC]
Y1: 通道号低字节
Y2: 通道号高字节

响应: [00][2A][Z1][00×15][CRC]
Z1: 0=未到位, 1=到位
```

##### 0x2B — 读一行到位开关

```
请求: [addr][2B][Y1][00×15][CRC]
Y1: 行号 (0-29)

响应: [00][2B][Z1-Z10][00×6][CRC]
Z1-Z10: 每列的到位状态 (bit0-bit7对应8个通道)
        共30列需要4字节，但M109E用10字节(100通道)
        本板需扩展或只返回前80通道
```

> ⚠️ **待讨论**：0x2B指令在30列时如何编码？建议用Z1-Z4(32bit)表示30列。

##### 0xFF — 设置从机地址

```
请求: [FF][01][00×16][CRC]  ← 设置为地址1
请求: [FF][02][00×16][CRC]  ← 设置为地址2
...
请求: [FF][08][00×16][CRC]  ← 设置为地址8

响应: 无（广播地址）
注意: 设置后指示灯闪烁次数=新地址
```

#### 3.4.5 通讯时序

```
主机发送指令
    ↓
等待 50ms
    ↓
从机响应（超时 1000ms）
    ↓
主机处理响应
    ↓
可发起下一条指令
```

#### 3.4.6 验收标准
- [ ] 所有指令均能正确收发

### 3.5 F05 — 电流检测（详细）

#### 3.5.1 硬件参数
- 采样电阻：R85 = 20mΩ（GND与DGND之间）
- 运放：OPA2365AIDR（U23.1），同相放大器
- 输入电阻：R99 = 1kΩ
- 反馈电阻：R100 = 20kΩ
- **增益 = 1 + 20k/1k = 21倍**
- ADC串联保护：R108 = 10Ω
- 钳位二极管：D1 = 1N5819HW
- ADC：12bit，3.3V参考，通道ADCX_IN0 (PA0)

#### 3.5.2 换算公式

```
V_shunt = I_load × 0.020Ω
V_out   = V_shunt × 21 = I_load × 0.42 V/A
ADC值   = V_out / 3.3 × 4096

I_load(mA) = ADC值 × 1.91

参考值：
  1A  → ADC ≈ 524
  2A  → ADC ≈ 1048
  3A  → ADC ≈ 1572
  5A  → ADC ≈ 2620
```

#### 3.5.3 接口定义

```c
/**
 * @brief 电流检测模块初始化
 */
void ADC_Sense_Init(void);

/**
 * @brief 启动一次ADC采样（电机运行期间周期调用）
 * @note  建议每10ms采样一次
 */
void ADC_Sense_Start(void);

/**
 * @brief 获取当前电流值
 * @return 电流值(mA)
 */
uint16_t ADC_Sense_GetCurrent(void);

/**
 * @brief 获取本次运行的峰值电流
 * @return 峰值电流(mA)
 */
uint16_t ADC_Sense_GetPeak(void);
```

#### 3.5.4 电流判定阈值

| 场景 | 阈值 | ADC阈值 | 说明 |
|------|------|---------|------|
| 过流 | >3250mA | ADC > 1701 | AO4812降额上限(70°C×50%) |
| 过流保护 | >4000mA | ADC > 2094 | 硬件极限保护 |
| 欠流 | <50mA | ADC < 26 | 返回错误码0x02 |
| 正常 | 50-3250mA | ADC 26-1701 | 正常范围 |

> **AO4812降额依据**：额定8.5A(25°C)/6.5A(70°C)，按50%降额 → 工作上限3.25A。
> 阈值需要实际电机测试后微调，以上为初始计算值。

#### 3.5.4 验收标准
- [ ] 静态电流读数稳定（无电机运行时接近0mA）
- [ ] 电机运行时电流读数与万用表偏差<10%
- [ ] 过流/欠流判定正确

### 3.6 F07 — 光幕控制（详细）

#### 3.6.1 硬件连接
- 光幕1：INPUT_11 (PA12) — NPN型光幕传感器
- 光幕2：INPUT_12 (PA11) — 备用光幕
- 光幕供电：需外部供电（本板不控制光幕电源）

#### 3.6.2 信号逻辑
- **常态（无掉货）**：NPN截止 → 4.7kΩ上拉至+24V → 光耦截止 → MCU读到**高电平**
- **掉货检测**：NPN导通 → 拉低 → 光耦导通 → MCU读到**低电平**
- **有效电平**：**低电平 = 检测到掉货**

```
无掉货:  PA12/PA11 = HIGH (1)
有掉货:  PA12/PA11 = LOW  (0)
```

#### 3.6.3 工作模式（兼容M109E）

| 模式 | 说明 |
|------|------|
| 0 | 电机运转不参考光幕 |
| 1 | 启动前光幕自检→自检通过才启动→到位后停止 |
| 2 | 启动前光幕自检→自检通过才启动→掉货检测到立即停止→未检测到则超时停止 |

#### 3.6.4 光幕自检逻辑

```
自检流程:
1. 读取INPUT_11/INPUT_12状态
2. 如果为低电平（有遮挡/有掉货）→ 自检失败 → 返回错误码0x04
3. 如果为高电平（无遮挡）→ 自检通过 → 启动电机

掉货检测（模式2）:
1. 电机运行中持续监测INPUT_11/INPUT_12
2. 检测到低电平（NPN导通=掉货）→ 记录遮挡时间 → 立即停止电机
3. 遮挡时间 > 0 表示有掉货
```

#### 3.6.5 验收标准
- [ ] 模式0：光幕状态不影响电机运行
- [ ] 模式1：光幕为低电平（掉货）时自检失败，返回0x04
- [ ] 模式2：电机运行中检测到低电平后立即停止
- [ ] 遮挡计时准确
- [ ] 常态（高电平）时不会误触发
