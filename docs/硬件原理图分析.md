### 1.2 矩阵驱动 — 74HC595 移位寄存器

共使用 **8片 74HC595D** 级联（P端4片 + N端4片）：

**P端（行驱动/正极）— 4片级联：**
| 芯片 | 输出 | 信号名 | 用途 |
|------|------|--------|------|
| U1 | Q0-Q7 | P0-P7 | 行 0-7 |
| U2 | Q0-Q7 | P8-P15 | 行 8-15 |
| U3 | Q0-Q7 | P16-P23 | 行 16-23 |
| U4 | Q0-Q7 | P24-P31 | 行 24-31（P30/P31未用） |

控制信号：`IOP_595_DS`, `IOP_595_SHCP`, `IOP_595_STCP`, `IOP_595_MR`

**N端（列驱动/负极）— 4片级联：**
| 芯片 | 输出 | 信号名 | 用途 |
|------|------|--------|------|
| U5 | Q0-Q7 | N0-N7 | 列 0-7 |
| U?? | Q0-Q7 | N8-N15 | 列 8-15（编号PDF未抓到） |
| U26 | Q0-Q7 | N16-N23 | 列 16-23 |
| U27 | Q0-Q7 | N24-N31 | 列 24-31（N30/N31未用） |

控制信号：`ION_595_DS`, `ION_595_SHCP`, `ION_595_STCP`, `ION_595_MR`

### 1.3 电平转换

**SN74LVC8T245PWR (U9)** — 8通道双向电平转换器
- VCCA = 3.3V（MCU侧）
- VCCB = 5V（595侧）
- 传输方向：DIR引脚控制
- 用于：P/N两组595的控制信号（DS, SHCP, STCP, MR）

### 1.4 每通道电机驱动电路

**P端（正极驱动）— 30路：**
```
MCU/595输出 → 1kΩ电阻 → SS8050(NPN)基极
                            ↓
SS8050 集电极 → 4953(P-MOSFET对管)栅极
                    ↓
4953 源极 = +24V, 漏极 = PM_OUTx（输出到电机正极）
SS8050 发射极 = GND
10kΩ 下拉电阻（基极到GND）
1N5819HW 续流二极管（电机两端）
NCD0603Y2 黄色LED指示灯
```

**N端（负极驱动）— 30路：**
```
MCU/595输出 → 1kΩ电阻 → AO4812(Dual N-MOS)栅极
                            ↓
AO4812 源极 = GND, 漏极 = NM_OUTx（输出到电机负极）
10kΩ 下拉电阻（栅极到GND）
1N5819HW 续流二极管
```

### 1.5 电流检测电路

**OPA2365AIDR (U23)** — 双路运放

**U23.1 — 电流检测放大器：**
- R85 = 20mΩ 采样电阻（GND与DGND之间）
- R99 = 1kΩ 输入电阻
- R100 = 20kΩ 反馈电阻
- R96 = 0Ω 跳线
- **增益 = 1 + 20k/1k = 21倍**
- 输出经R108(10Ω)保护后到ADCX_IN0 (PA0)
- D1 = 1N5819HW 钳位二极管保护ADC

换算公式：
```
I_load(mA) = ADC值 × 1.91
参考：1A≈524, 2A≈1048, 3A≈1572
过流阈值(2.5A): ADC>1310
欠流阈值(50mA): ADC<26
```

**U23.2 — NTC温度缓冲器：**
- CN12 (3pin) 外接NTC传感器
- R161 = 100kΩ 下拉/分压电阻
- 输出到ADCX_IN1 (PA1)

### 1.6 光耦输入 — 12路

**LTV-247** 四通道光耦 × 3片 (U18, U19, U20)
- 输入：IN1-IN12（经SMF26CA TVS保护 + 4.7kΩ上拉到+24V）
- 输出：INPUT_1 ~ INPUT_12（到MCU GPIO）
- 有效电平：**低电平有效**（NPN传感器导通时拉低）

### 1.7 RS-485 通讯

**TP8485E-SR (U24)**
- 波特率：9600bps（由MCU USART配置）
- 120Ω终端电阻（R189）
- 360Ω上下拉电阻（R187, R191）
- SMF6.5CA TVS保护（D69, D70, D72）
- 连接器：DB2ERC-3.81-3P-GN × 2（A/B/GND）

### 1.8 继电器/NMOS输出 — 4路

**EL357C** 光耦隔离 × 4片
- 控制信号：io_nmos1 ~ io_nmos4（来自MCU GPIO）
- 输出：NMOS1 ~ NMOS4
- 带AO4812 NMOS开关
- 连接器：CN13 (WJ15EDGRC-3.81-04P)

### 1.9 电源

```
DC 24V 输入
    ↓
LM2596S-5.0 (U31) → +5V（为595、光耦等供电）
    ↓
AMS1117-3.3 (U30) → VCC 3.3V（为MCU供电）

保护元件：
- SS34/SS54 肖特基二极管（防反接）
- SMAJ5.0CA / SMAJ3.3A TVS（过压保护）
- SMF26CA TVS（24V输入保护）
- 470μF + 330μF + 100μF 电解电容滤波
```

## 三、关键发现与注意事项

### 3.1 ⚠️ RAM严重不足
- STM32F103C8T6 只有 **20KB RAM**
- 如果存储900通道状态表（每通道10字节），需要 **9KB**
- 加上栈、全局变量、缓冲区，RAM非常紧张
- **解决方案**：
  1. 精简状态结构体（每通道压缩到4-5字节）
  2. 使用Flash存储电机类型配置（不常变化）
  3. 运行时只保留当前运行通道的状态
  4. 使用 `__attribute__((packed))` 减少对齐浪费

### 3.2 ⚠️ Flash也需要节省
- 64KB Flash，固件代码+配置存储+OTA需要精心规划
- 建议预留 4KB 用于参数存储（EEPROM模拟）
- OTA需要双bank的话空间不够，建议只支持单bank+bootloader

### 3.3 595驱动方式
- P端和N端各有独立的 DS/SHCP/STCP/MR 信号
- 通过 SN74LVC8T245 电平转换（3.3V→5V）
- 可以用SPI或GPIO位操作驱动
- SHCP频率最高约20MHz（74HC595规格）
- 建议使用 **SPI+DMA** 驱动，效率最高

### 3.4 电流检测精度
- 20mΩ采样电阻 + OPA2365运放
- 需要确认增益设置（看电阻值）
- ADC 12bit，3.3V参考，需要计算分辨率
- 建议软件做滑动平均滤波

### 3.5 光耦输入
- 12路光耦输入（LTV-247），用于到位检测
- 有效高/有效低需要确认光耦接法
- 看原理图是+24V通过4.7kΩ上拉，光耦导通时输入为低

### 3.6 实际矩阵规模
- 原理图标题为"货道板30*10"
- P端30路（PM_OUT1-30），N端30路（NM_OUT1-30）
- 理论上 30×30 = 900 通道
- 但实际可能是 **30行×10列 = 300通道**（3块板级联=900）
- 需要确认：单板驱动300还是900通道？

### 3.7 串口资源
- USART1 用于RS-485通讯（与上位机）
- USART2, USART3 未使用（可用于调试/扩展）
- 建议保留一路UART用于调试打印

## 五、固件开发的关键约束

1. **RAM只有20KB** — 状态表必须精简，不能像M109E那样每个通道存10字节
2. **Flash只有64KB** — 代码量需要控制，不能加太多功能
3. **GPIO有限** — 595的4线控制信号已经占了8个GPIO
4. **单UART** — 只有USART1用于通讯，调试需要用SWO或segger RTT
5. **ADC双通道** — 电流检测占用PA0/PA1
6. **没有硬件SPI给595** — 如果SPI1(PA5/PA6/PA7)已被用作GPIO，则需要位操作驱动595
